Crea un formulario para con todos los campos de desarrollador (menos id)
	- Divide el formulario en 2 formularios distintos
		- El primero contendrá toda la información personal del desarrollador
		- El segundo todos los datos laborales
			- Fecha_alta, activo e id_estudio
		- Id_estudio debe ser un select que contenga todas las opciones actuales de estudio y la 		opción "Sin estudio", que mandaría NULL a la base de datos.
		- Id_estudio solo se muestra si el check de activo está marcado.
		- Activo debe ser un checkbox
		- Limita los campos a sus dimensiones marcadas por la base de datos
		- Limpia caracteres no permitidos


Ampliación:
	- Implementa un sistema de validación como en el ejemplo
		- nombre obligatorio y longitud mínima 2.
		- apellido obligatorio y longitud mínima 2.
		- email opcional, pero si está relleno debe cumplir:
			- contiene @
		- ciudad opcional (si está rellena, mínimo 2 caracteres).
		- pais opcional (si está relleno, mínimo 2 caracteres).
		- fecha_nacimiento opcional, pero si está rellena no puede ser una fecha futura.
		- activo es checkbox:
			- si está marcado:
				-fecha_alta obligatoria	
				- id_estudio obligatorio (no puede ser “Sin estudio”)
			- si NO está marcado:
				- id_estudio debe quedar oculto (como ya tenéis)
				- id_estudio debe enviarse como vacío/NULL (simulado)
				- fecha_alta no puede ser futura.
	- En caso de error se tiene que mostrar mensaje y actualizar el css de cada componente.
	- Al enviar el formulario el foco tiene que irse al primer error
	- Al iniciar y resetear el formulario se tiene que poner el foco en el primer campo.
